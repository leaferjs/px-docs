# PxGrow 通用插件需求申请机制

## 🌟 一、机制说明

PxGrow 插件基于 **用户真实需求驱动开发**。

当插件需求达到最低支持门槛或具备战略价值时，官方启动开发，并采用 **分阶段定价机制**：

- 🥇 众筹阶段：支持价
- 🥈 Beta 阶段：早鸟价
- 🥉 正式发布：销售价

不同阶段对应不同风险与确定性，价格逐步上调。

## 📝 二、用户申请阶段

### 1️⃣ 申请方式

用户可联系我们沟通插件需求，并提交申请表单。

<img src="https://www.leaferjs.com/image/qrcode/leafer.jpg" width="133" style="border-radius: 6px; box-shadow: 0 0px 50px rgba(0,0,0, 0.05);">

### 2️⃣ 申请表单字段建议

- 🏷 插件名称（可选暂定名）
- ✨ 功能描述（核心需求、操作场景、对标产品或演示链接）
- 🌐 使用场景（Web / 移动 / 桌面）
- 👤 用户类型（个人 / 团队 / 企业）
- ⭐ 优先级说明（可选）

## 📊 三、立项与门槛规则

插件进入众筹阶段需满足以下条件之一：

- 达到最低支持人数
- 或达到最低众筹金额
- 或具备明确战略价值（由官方评估）

### 💡 退款说明

- 若众筹未达标，将全额退款
- 若已达标并正式立项，进入开发流程

## 🛠 四、内部可行性评估

在正式立项前，官方将进行评估：

### 1️⃣ 技术评估

- 🔌 是否与现有引擎兼容
- ⚙️ 技术实现难度与开发周期
- 💰 所需资源投入

### 2️⃣ 产品价值评估

- 🌍 用户覆盖率与潜在影响力
- 🧩 与现有插件生态的组合价值
- 🚀 战略意义与长期价值

### 3️⃣ 风险与优先级排序

- ⚠️ 对风险高或重复插件进行二次确认
- 🔝 优先级高的插件优先开发

## 🏁 五、众筹阶段

当插件进入众筹阶段：

### 公布信息包括：

- 🏷 插件名称与功能概述
- ⏳ 预计开发周期
- 💰 正式原价与当前众筹价
- 👑 众筹支持者权益说明
- ⌚ 众筹截止时间或人数门槛

### 众筹支持者权益：

- 优先使用插件
- 优先参与内测
- 优先反馈通道

## ⚡ 六、Beta 阶段

Beta 版本上线后，进入早鸟阶段：

- 🌐 提供可用测试版本
- 📄 公布更新日志
- 📚 提供基础使用文档

## 🎉 七、正式发布阶段

插件正式发布后：

- 🌐 发布稳定版本
- 📚 完整文档与示例
- 🔄 正常维护与版本迭代

正式版价格为标准商业价格。

> 正式价格可能根据功能完整度与版本能力进行调整。

## 🔄 八、后续迭代与维护

- 📈 收集正式用户反馈
- 🖌 持续优化体验
- 🔧 修复问题与功能增强

## ⚠️ 九、注意事项

### 免责声明

- 插件开发基于需求驱动，不保证所有申请都会立项
- 开发周期视技术难度与资源情况调整
- 价格阶段变动以官方公告为准
